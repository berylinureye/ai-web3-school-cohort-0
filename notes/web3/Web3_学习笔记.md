# Web3 学习笔记：密码学、钱包、EOA、交易、Gas 与智能合约

## 页面记录与理解

### 密码学（Cryptography）
- **讲什么？**  密码学页面解释了区块链体系的基础: 哈希、私钥、公钥、签名和 Merkle 树等概念。
- **第一次听到的词**  哈希、私钥、公钥、签名、Merkle 树。
- **和钱包/交易/链的关系**  私钥控制钱包资产，签名授权交易；哈希验证数据完整性；Merkle 树帮助轻钱包验证交易。
- **比喻理解**  两层保险柜：内层存钱，外层/中间层公开接收资金，签名授权钱从内层流向中间层。

### 钱包与 EOA
- **钱包是什么？**  钱包是操作内层保险柜的管理器，生成私钥、公钥/地址、签名，显示资产。钱包本身不存钱。
- **EOA (Externally Owned Account)**  链上由私钥控制的账户，能发起交易和签名，是资产真正被控制的身份。
- **交易与 Gas**  交易是操作资产的指令；Gas 是节点处理交易的费用，支付 Gas 才能执行改变链上状态的操作。签名消息不消耗 Gas，仅验证身份。
- **比喻**  钱包 + EOA = 遥控器+钥匙；交易 = 填写包裹申请单；签名 = 授权印章；Gas = 支付快递员运费；节点 = 快递员搬运资产；事件 = 快递回执。

### 智能合约（Smart Contract）
- **讲什么？**  合约是部署在链上的程序，包含规则和状态，可自动执行操作。
- **第一次听到的词**  Solidity、EVM、ABI、Event、Upgrade。
- **和钱包/交易/链的关系**
  - **Solidity**：合约语言，钱包发送交易由 EVM 执行字节码。
  - **ABI**：前端和钱包编码调用函数和参数。
  - **Event**：链上日志，前端监听更新 UI。
  - **Upgrade**：合约升级机制影响用户权限和安全。
- **比喻理解**  智能合约 = 银行柜台自动操作规则；你发起交易 = 提交申请；签名 = 授权印章；Gas = 支付处理费；节点 = 柜台执行逻辑；事件 = 回执通知前端。
- **调用流程**
  1. 前端构造交易数据。
  2. 钱包弹窗用户签名。
  3. 广播交易到节点。
  4. EVM 执行合约逻辑，更新状态，发出事件。
  5. 前端监听事件，更新显示。
- **最小实践**  选择简单的 ERC‑20 或 NFT 合约，区分读写函数，查看事件和权限函数，总结风险点。
