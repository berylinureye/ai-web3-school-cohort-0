## 卡片 1 · EOA（外部账户）

- **本质**：基于公私钥对的链上账户。"持有私钥的人 = 账户的主人"
- **例子**：MetaMask 默认创建的账户。地址形如 `0xb3b69AFA...3983`
- **安全提醒**：私钥 / 助记词一旦泄漏 = 资产全失。**all-or-nothing** 安全模型
- **常见误区**：以为"账户在 MetaMask 里" → 实际上账户在链上，MetaMask 只是管理私钥的 UI

## 卡片 2 · 钱包（Wallet）

- **本质**：管理私钥的工具，**不是"存币的地方"**。币始终在链上，钱包只是签名工具
- **例子**：MetaMask（浏览器扩展）、Rainbow（移动端）、Ledger / Trezor（硬件钱包）
- **安全提醒**：装钱包扩展只从官方网站 / 应用商店；**核对 Publisher = "MetaMask Inc"**，防仿冒
- **常见误区**：以为"卸载钱包 = 失去币" → 实际上重装钱包 + 输入助记词，账户立刻恢复

## 卡片 3 · 助记词（Seed Phrase / Mnemonic）

- **本质**：12 / 24 个英文单词，是私钥的可读备份。**一份助记词管整棵账户树**（同助记词可派生无数个地址）
- **例子**：BIP-39 标准词表，比如 `apple banana cat ...`
- **安全提醒**：**只纸抄、离线保存**，永远不输入任何网页 / 截图 / 上传云盘。任何让你输入助记词的网站 100% 是钓鱼
- **常见误区**：以为"助记词加密存在电脑里安全" → 任何联网设备都有被窃风险

## 卡片 4 · 私钥（Private Key）

- **本质**：64 位十六进制字符串，账户控制权的数学根源。地址 = 由私钥推导出公钥再哈希后截取
- **例子**：`0xabcd1234...`（64 chars）
- **安全提醒**：private key 与助记词等价——任何持有它的人都能转走你的全部资产
- **常见误区**：以为"地址私密" → 地址是公开的（必须公开才能收款），**唯一私密的是私钥**

## 卡片 5 · 签名（Signature）

- **本质**：用私钥对数据做密码学加签，**任何人能用公钥验证"这签名来自这个地址"**
- **例子**：
  - `personal_sign` (EIP-191)：签任意字符串，证明地址所有权
  - `EIP-712 typed data`：签结构化 JSON
  - `Permit / Permit2`：EIP-712 的特化，签完等于 gasless 授权对方转走你的代币
- **安全提醒**：签名**不上链、不花 gas**，但 **EIP-712 / Permit 类签名等价于"未来交易的代笔授权书"**——钱包 UI 把"签名"显示得比"交易"轻量，但风险经常更高
- **常见误区**：以为"没花 gas = 没事" → 钓鱼组最喜欢用签名而不是直接发交易，因为更隐蔽

## 卡片 6 · 交易（Transaction）与 Gas

- **本质**：链上写操作的最小单位。任何"改变状态"的动作都要打包成 tx + 付 gas
- **例子**：从 Account 1 转 0.01 ETH 给 Account 2，或调用 Uniswap 的 swap 函数
- **安全提醒**：
  - tx 一旦上链**不可撤回**——To 地址、Value、Data 都要在钱包弹窗里逐项核对
  - Gas 即使交易失败也不退（EVM 已经算过了）
- **常见误区**：以为"撤销交易等下次就行" → 链上无 undo；EIP-1559 burn 的部分永久销毁

## 卡片 7 · 智能合约（Smart Contract）

- **本质**：部署到链上的程序。**不是"自动执行的法律合同"**，是带状态、有规则、所有人能调用的公共代码
- **例子**：Uniswap Router（撮合）、ERC20 token（余额表）、Gnosis Safe（多签钱包）
- **安全提醒**：合约部署后基本改不动（除非有 upgrade proxy）；**bug 即公共风险**
- **常见误区**：把"合约审计过"当成"完全安全" → 审计只覆盖审计时的代码 + 已知攻击面，新漏洞 / 升级权限滥用 / 治理被攻击都不在审计范围

## 卡片 8 · 合约账户 vs EOA

- **本质**：链上账户分两类——EOA（公私钥控制）和合约账户（部署时写死的代码控制）
- **例子**：
  - EOA：MetaMask 默认账户
  - 合约账户：Uniswap Router / Gnosis Safe / 一个 ERC20 token 合约
- **安全提醒**：合约账户**没有私钥**——它的行为由部署时的代码决定。要看清"谁能 upgrade / pause / withdraw"
- **常见误区**：以为"合约就是 dApp" → 合约只是 dApp 的"后端"。前端 UI / RPC / 索引器都是其他组件

## 卡片 9 · EVM（Ethereum Virtual Machine）

- **本质**：以太坊系链的"执行核心"——给定一笔交易，按规则跑、改状态、扣 gas
- **例子**：以太坊主网 / Sepolia / Arbitrum / Base / Optimism 都用 EVM
- **安全提醒**：EVM 只管"执行层"，**共识/出块/链怎么连**是其他层的事
- **常见误区**：以为"EVM = 区块链" → EVM 只是其中一层。Bitcoin 没 EVM；Solana 用 SVM 不用 EVM

## 卡片 10 · 测试网 vs 主网（Sepolia vs Mainnet）

- **本质**：测试网是真正运行的独立区块链，只是币没价值——为开发者调试用
- **例子**：Sepolia (Chain ID 11155111) 是当前以太坊推荐的测试网
- **安全提醒**：**测试网钱包永不打主网真币**——给练习钱包定明确边界
- **常见误区**：以为"测试网交易不是真的" → 测试网交易完全真实地写进区块链，只是 ETH 单位无价值

## 为什么"私钥 / 助记词 / 签名 / 授权"需要特别谨慎

这四类信息**等级递增的危险性**来自同一个根源：**链上世界没有撤销键**。

| | 危险性 | 一旦失守的后果 |
|---|---|---|
| **私钥** | ★★★★★ | 持有者立即获得资产完全控制权，可瞬间清空账户 |
| **助记词** | ★★★★★ | 等同于私钥（甚至更强，能推导出整棵账户树） |
| **签名（特别是 EIP-712 / Permit）** | ★★★★ | 不立即损失，但等同"未来交易代笔授权书"，攻击者可在 deadline 内随时转走指定资产 |
| **授权（approve / setApprovalForAll）** | ★★★ | 已上链的 allowance，被授权方可以无声转走对应资产；需手动 revoke 才停止 |

**操作纪律**：

1. 私钥 / 助记词 **永不数字化**（不截图、不云盘、不输入网页、不发任何消息）
2. 签名**逐字读 message 内容**——尤其当 `domain.verifyingContract` 或 `spender` 字段陌生时一律拒签
3. 授权**用完即 revoke**（[revoke.cash](https://revoke.cash/) 或 Etherscan 的 Token Approval Checker）
4. 给练习钱包定边界：**测试网练手 + 主网小额 + 大额单独硬件冷钱包**，三套分离
