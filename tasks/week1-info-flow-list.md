## 进阶部分：5 条高质量内容深读

### 内容 1: **Vitalik Buterin "The Promise and Challenges of Crypto + AI Applications"**

- **来源**: https://vitalik.eth.limo/general/2024/01/30/cryptoai.html
- **作者**: @VitalikButerin
- **发表时间**: 2024-01-30
- **类型**: 长博客

#### 讨论的核心问题
Crypto 和 AI 的 4 种结合方式：
1. AI 作为 game 的玩家（自动化交易、套利、参与协议）
2. AI 作为游戏接口（语义化的钱包/dApp 操作）
3. AI 作为 game rules（链上 AI 模型作为治理决策者）
4. AI 作为 game objective（DAO 训练共享 AI）

#### 背后的技术 / 产品趋势
- AI 的 hallucination 在链上有特殊危险性（不可撤销操作 + 错误 confident）
- "可验证 AI"（zk-ML / TEE / consensus on ML output）是未来 5 年的关键技术问题
- **Agent 的安全边界设计**（什么能做什么不能做）跟我课程 Day 2 学的"AI Agent 红线"一脉相承

#### 跟 Week 1 课程主题的关系
Week 1 让我们建立"AI 跟 Web3 共同语言"——Vitalik 这篇是**这种语言的最高权威版本**，他亲自把 AI × Web3 的 design space 切成 4 块，比任何 cohort 课程 framework 都清晰。

#### 我还不理解的问题
- "AI 作为 game rules" 那一节涉及 zk-ML，但我对 zero-knowledge proof 还完全不懂——后面如果有 ZK 课要补
- "Adversarial ML" 这个 buzz 词在链上场景下具体怎么实现？

### 内容 2: **Andrej Karpathy "[1hr Talk] Intro to Large Language Models"**

- **来源**: https://youtu.be/zjkBMFhNj_g
- **作者**: @karpathy
- **发表时间**: 2023-11
- **类型**: 1 小时视频讲座

#### 讨论的核心问题
LLM 的工作机制全栈拆解：
- pretraining: 在互联网文本上 next-token prediction
- finetuning: 用人工标注的 instruction-response 对调教模型变成 assistant
- alignment: RLHF / Constitutional AI 让模型对齐人类偏好
- security: prompt injection / jailbreak / data poisoning / RAG attack

#### 背后的技术 / 产品趋势
- LLM 不是"知识库"，是"压缩了互联网的概率分布"
- LLM 安全是个完全新的领域，跟传统软件安全完全不一样
- Agent harness 是后 LLM 时代的关键能力——单纯模型能力提升后，怎么"约束 + 协调"模型才是产品胜负手

#### 跟 Week 1 课程主题的关系
我做 #8 AI 概念卡片的"四层边界"框架（prompt / context / agent / tool）—— **直接来源于这套 Karpathy 的心智模型**。他对 LLM 工作方式的描述影响了行业 2024-2025 的所有 agent 框架设计。

#### 我还不理解的问题
- RLHF 跟 Constitutional AI 在数学上具体怎么不同？
- "model self-improvement"（模型用自己的输出训练自己）这条路线在 2025 的进展？

#### 跟 Week 1 课程主题的关系

### 内容 5: **OpenZeppelin 2024 Security Incident Recap**

- **来源**: https://blog.openzeppelin.com/category/security/
- **作者**: @OpenZeppelin
- **类型**: 持续更新的攻击事件分析

#### 讨论的核心问题
2024 年发生的所有重大智能合约攻击案例：
- Curve Finance reentrancy via Vyper compiler bug
- Stake.com hot wallet 私钥泄漏
- Mixin Network supply chain attack
- KyberSwap 复杂的 oracle manipulation
- WazirX 多签被钓鱼

#### 背后的技术 / 产品趋势
- **私钥管理仍然是最脆弱的环节**——大部分大额失窃跟合约 bug 无关，而是 ops 失败（hot wallet 泄漏、操作员被钓鱼）
- **依赖供应链攻击**（编译器 bug、库版本 backdoor）变得越来越常见
- **复杂的协议（oracle / cross-protocol composition）** 比简单协议更容易出事——简单 = 安全

#### 跟 Week 1 课程主题的关系
我 0519 学的"#4 把 owner 当永远可信"（rug pull 根源）+ 私钥/签名/授权的安全教育——这些不是抽象警告，**2024 年每个月都在发生真实事故，单次损失千万美元以上**。这给我做产品研究时的"信任尽调表"提供了真实素材。
